# Enable-OSDeployMDT

## Purpose

`Enable-OSDeployMDT` initializes an existing MDT Deployment Share for use with the OSDeployMDT and OSDCloud toolchain. It is a one-time setup function that configures the share so that `Build-OSDeployMDT` runs automatically during **Update Deployment Share**, injects WinPE components, and enables PowerShell Gallery and OSDCloud support inside the boot image.

> Requires local **Administrator** rights.

***

## What It Does

### 1. Resolve MDT Installation Directory

Calls the private `Get-MDTEnvINSTALLDIR` function to locate the MDT installation path (`INSTALLDIR`, typically `C:\Program Files\Microsoft Deployment Toolkit`). Aborts if MDT is not installed.

### 2. Select the Active Deployment Share

Calls `Select-MdtDeploymentShare` to prompt the user (or auto-detect) the target MDT Deployment Share (`DEPLOYROOT`). Aborts if no share is selected.

### 3. Create Template Folder Structure

Ensures the following folders exist under `DEPLOYROOT`, creating them if missing:

| Folder                        | Purpose                                   |
| ----------------------------- | ----------------------------------------- |
| `Templates\`                  | Holds customized MDT template files       |
| `Templates\winpe-driver\`     | Drop WinPE-injected drivers here          |
| `Templates\winpe-extrafiles\` | Drop extra files to include in WinPE here |

### 4. Copy Template Files from INSTALLDIR

Copies the following files from `INSTALLDIR\Templates` to `DEPLOYROOT\Templates` (skips if the destination already exists):

| File                  | Purpose                                  |
| --------------------- | ---------------------------------------- |
| `winpeshl.ini`        | WinPE shell launcher configuration       |
| `Wimscript.ini`       | WIM exclusion list for capture           |
| `Unattend_PE_x64.xml` | Unattend file applied during WinPE boot  |
| `LiteTouchPE.xml`     | MDT WinPE definition (components, exits) |

### 5. Copy WinPE Wallpaper

Copies `INSTALLDIR\Samples\Background.bmp` to `DEPLOYROOT\Templates\background.bmp` (skips if already present). This file can be replaced with a custom wallpaper.

### 6. Rewrite Path References in LiteTouchPE.xml

Updates `DEPLOYROOT\Templates\LiteTouchPE.xml` to replace `%INSTALLDIR%` references with `%DEPLOYROOT%` so that the customized copies are used instead of the originals:

| Before                                              | After                                               |
| --------------------------------------------------- | --------------------------------------------------- |
| `%INSTALLDIR%\Templates\Unattend_PE_%PLATFORM%.xml` | `%DEPLOYROOT%\Templates\Unattend_PE_%PLATFORM%.xml` |
| `%INSTALLDIR%\Templates\winpeshl.ini`               | `%DEPLOYROOT%\Templates\winpeshl.ini`               |

### 7. Sync WinPE Components

Merges the `<Components>` element from `DEPLOYROOT\Boot\LiteTouchPE_x64.xml` (the live boot XML generated by MDT) into `DEPLOYROOT\Templates\LiteTouchPE.xml`.

If `Boot\LiteTouchPE_x64.xml` does not yet exist (share has never been updated), a default set of WinPE optional components is written instead:

`winpe-scripting`, `winpe-hta`, `winpe-wmi`, `winpe-securestartup`, `winpe-fmapi`, `winpe-dismcmdlets`, `winpe-dot3svc`, `winpe-enhancedstorage`, `winpe-hsp-driver`, `winpe-netfx`, `winpe-platformid`, `winpe-pmemcmdlets`, `winpe-powershell`, `winpe-pppoe`, `winpe-rndis`, `winpe-securebootcmdlets`, `winpe-storagewmi`, `winpe-wds-tools`

### 8. Register Build-OSDeployMDT Exit

Ensures the following `<Exit>` entry is present in `DEPLOYROOT\Templates\LiteTouchPE.xml`:

```xml
<Exit>start /wait pwsh.exe -ExecutionPolicy Bypass -Command "Build-OSDeployMDT"</Exit>
```

This causes `Build-OSDeployMDT` to run automatically every time MDT regenerates the boot image via **Update Deployment Share**.

### 9. Update Control\Settings.xml

Creates a backup of `DEPLOYROOT\Control\Settings.xml` (once, no overwrite), then applies the following changes:

| Setting                              | Old Value                             | New Value                                 |
| ------------------------------------ | ------------------------------------- | ----------------------------------------- |
| Background image path                | `%INSTALLDIR%\Samples\Background.bmp` | `%DEPLOYROOT%\Templates\background.bmp`   |
| `SupportX86`                         | `True`                                | `False`                                   |
| `Boot.x86.UseBootWim`                | `True`                                | `False`                                   |
| `Boot.x86.IncludeAllDrivers`         | `True`                                | `False`                                   |
| `Boot.x86.IncludeNetworkDrivers`     | `True`                                | `False`                                   |
| `Boot.x86.IncludeMassStorageDrivers` | `True`                                | `False`                                   |
| `Boot.x86.IncludeVideoDrivers`       | `True`                                | `False`                                   |
| `Boot.x86.IncludeSystemDrivers`      | `True`                                | `False`                                   |
| `Boot.x86.GenerateLiteTouchISO`      | `True`                                | `False`                                   |
| `Boot.x64.ExtraDirectory`            | _(empty)_                             | `%DEPLOYROOT%\Templates\winpe-extrafiles` |
| `Boot.x64.ScratchSpace`              | `32`                                  | `512`                                     |

x86 boot image generation is disabled (x64-only). The scratch space is increased to 512 MB for better compatibility with modern WinPE workloads.

***

## End State

After a successful run, the deployment share is configured so that the next **Update Deployment Share** operation will:

* Inject the customized WinPE components and extra files
* Display the custom wallpaper
* Launch `Build-OSDeployMDT` on WinPE exit, enabling:
  * PowerShell Gallery access inside the boot image
  * OSDCloud deployment via `Deploy-OSDCloud`

```
DEPLOYROOT\
  Templates\
    LiteTouchPE.xml          # customized WinPE definition
    winpeshl.ini             # WinPE shell config
    wimscript.ini            # WIM capture exclusion list
    Unattend_PE_x64.xml      # WinPE unattend
    background.bmp           # wallpaper (replaceable)
    winpe-driver\            # inject WinPE drivers here
    winpe-extrafiles\        # inject extra WinPE files here
  Control\
    Settings.xml             # updated (x64-only, 512 MB scratch)
    Settings.xml.backup      # original backup
```

***

## Example

```powershell
Enable-OSDeployMDT
```

Run once after creating a new MDT Deployment Share, before the first **Update Deployment Share**.
